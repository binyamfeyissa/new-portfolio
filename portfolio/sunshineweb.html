Project description

The Compound Management System (CMS) is a full-featured web platform designed to manage day-to-day operations in residential compounds, built under Tera, a software company. The system centralizes services like resident management, announcements, payments, maintenance requests, and internal communication between administrators, security teams, maintenance crews, and board members. I built the web front end from scratch and led the UI/UX decisions, ensuring the platform was both visually intuitive and functionally scalable. This system was deployed across multiple compounds in Addis Ababa and serves over 5,000 residents, streamlining processes that were previously done manually.

I personally handled:

  • Designing and implementing role-based dashboards for five distinct user types (admin, board members, guards, residents, maintenance teams).
  • Integrating real-time service request handling, announcement publishing, and internal communications with Firestores realtime capabilities.
  • Building complex form flows for staff scheduling, resident registration, and payment tracking.
  • Leading the UI consistency and experience for cross-role usability, making sure workflows remain smooth regardless of user type.
  • Architecting a fully responsive interface using React.js, Tailwind CSS, and Firebase Auth, making the system accessible on any device.

Impact of my role

The web platform replaced 90% of the previous manual record-keeping processes and reduced service request resolution time by over 80%. By bringing transparency, accountability, and automation to compound management, this platform significantly increased operational efficiency. As the front-end lead, I ensured each feature was tailored to real-world compound needs and built for scale.
